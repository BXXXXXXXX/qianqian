# Goal

定义 KTOS 当前阶段的产品聚焦原则，避免功能发散。

---

# Mission

Project One Hour：每天还老师一小时。

---

# Product Filter

以后讨论任何功能，都先问：

这个功能，真的能让老师今天早下班吗？

如果不能，就先不做。

---

# Core Problem

老师真正损耗的不是单一文书动作，而是反复在多个工具之间切换：

- 微信
- 备忘录
- Word
- Excel
- WPS
- ChatGPT

KTOS 要做的是把这些零散动作收回来。

---

# Core Principle

老师永远只描述事实，不再负责组织语言。

系统基于同一份事实记录，自动生成：

- 家长沟通草稿
- 成长档案素材
- 总结素材
- 待办与提醒

---

# Three Priority Capabilities

## 1. Communication Assistant

重点不是“AI 写消息”，而是降低沟通风险、提高表达稳定性。

## 2. Growth Memory

重点不是“手动建档”，而是让档案自然形成，并形成长期可追踪的成长记忆。

## 3. Today’s Inbox

重点不是功能展示，而是告诉老师今天先做什么、后做什么。

---

# Out of Scope

当前阶段不为了“看起来完整”而加入不能节省时间的功能入口。
